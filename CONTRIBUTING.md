# Contributing

## Setup

```bash
npm install
```

## Development workflow

1. Create a feature branch from `main`.
2. Add or update tests with your change.
3. Run `npm run quality:check` locally.
4. Open a pull request with a clear scope.

## Commit and PR expectations

- Keep commits focused and reviewable.
- Include test evidence in PR description.
- For breaking changes, update `docs/api-reference.md` and `CHANGELOG.md`.

## Reporting issues

Please use GitHub Issues and include:

- Node.js version
- Reproduction YAML/TS snippet
- Expected vs actual behavior
- Full error output
